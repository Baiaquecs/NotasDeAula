SELETORES

Os seletores ditam, na linguagem CSS, quem irá receber as instruções que foram dadas. Eles seguem o seu nome: SELECIONAM quem será estilizado. Já vimos sobre seletores no módulo passado, como os seletores de texto h1, h2 e h3 mas agora falaremos mais a fundo.

Seletor Universal *

O Seletor Universal, representado por *{}, atribui as suas regras a TODAS as tags HTML da página. É muito utilizado para zerar o MARGIN e PADDING.

Seletores de Texto (ou tag)

Os Seletores de Texto aplicam as suas regras pelo tipo de tag: assim, todos os elementos com a tag selecionada serão estilizados.

Exemplo:
  • h1{}: seleciona todas as tags "h1" da página.
  • p{}: seleciona todas as tags "p" da página.
  • div{}: seleciona todas as tags "div" da página.

Seletores de Classe

Os Seletores de Classe atribuem suas regras apenas aos elementos que possuem a class a que ele se refere. Esses seletores são identificados por sempre começarem com "." .

Exemplo:
  • .azul {}: seleciona todas as tags com a class="azul".
  • p.menu {}: seleciona apenas as tags "p" com a class="menu".
  • h2.post {}: seleciona apenas as tags "h2" com a class="post".

Observe que, apesar de na linguagem CSS, esses seletores serem identificados com o "." no início, o "." não precisa ser inserido no atributo "class" na linguagem HTML.
É também importante citar a possibilidade de especificar a que tipo de seletores de texto a class se refere - a colocando antes do "." -, visto que uma class pode ser utilizada infinitas vezes numa página HTML.

Seletor de Identidade

Os Seletores de Identidade atribuem suas regras apenas ao elemento com a ID a que ele se refere. Esses selotores são identicados por sempre começarem com "#".

Exemplo:
  • #logo {}: seleciona a tag com a id="logo"
  • #rodape {}: seleciona a tag com a id="rodape"
  • #pagina p {}: seleciona apenas os paragrafos dentro da tag com a id="pagina"

De maneira igual aos seletores de classe, a "#" não precisa ser inserida no atributo "id" na linguagem HTML. O atributo ID deve ser utilizado em apenas uma tag em toda a página HTML.

Seletor de Filho

Os Seletores de Filho selecionam apenas os elementos que são filhos (que estão IMEDIATAMENTE dentro) de outro elemento. Uma tag é filha da tag que está IMEDIATAMENTE "fora" dela. O símbolo que os representa é ">", mas há uma maneira especial de utilizá-lo.

Exemplo:
  • li>a {}: seleciona apenas as tag "a" que estão IMEDIATAMENTE dentro de uma tag "li".
  • div>p {}: seleciona apenas as tag "p" que estão IMEDIATAMENTE dentro de uma tag "div".
  • div>h2{}: seleciona as tag "h2" que estão IMEDIATAMENTE dentro de uma tag "div".

É importante frizar o "IMEDIATAMENTE dentro" porque o filho de uma tag são as tags que se abrem em sequência à ele. As tags dentro de uma tag filho, não são filhos do elemento principal.

Seletor de Descendente

Já os Seletores de Descendente selecionam todas as tags dentro deles, todos os seus "descentendes", e não apenas o "filho". Para identificar esses seletores, basta colocar a tag principal adicionar um espaço e, em seguida, a tag descendente.

Exemplo:
  • p a {}: seleciona todas as tag "a" que estejam dentro de "p", sendo ou não filho direto.

Seletor de Irmão Adjacente

Seleciona o elemento irmão que vem IMEDIATAMENTE após o principal. É caracterizado pelo símbolo "+".

Exemplo:
  • h1+p {}: seleciona a primeira tag "p" depois de uma tag "h1".

Seletor de Irmão Geral

Seleciona todos os elementos que são irmãos do principal - que venham depois dele-, não apenas a imediatamente após.

Exemplo:
  • h1~p {}: seleciona todas as tags "p" irmãs de "h1" - ou seja, que vem após a tag "h1".

Herança e Especificidade

São dois conceitos intrísecos ao CSS, muito importantes para entender o funcionamento da linguagem.

Herança

O conceito de herança rege o comportamento de alguns elementos de herdar algumas regras atribuídas aos elementos externos a eles. Um exemplo é a propriedade "color" no seletor "body", que atribui a cor escolhida a todos os elementos dentro do "body".

Especificidade

Já o comportamento de especificidade rege a ordem na qual as regras serão aplicadas. Os seletores possuem uma precedência que define quais regras serão aplicadas primeiro.
Em um caso, por exemplo, de dois seletores iguais, as regras do último escrito (o mais abaixo) é que será aplicado, COM EXCEÇÃO no caso da utilização do atributo CSS "!important" que define o seletor selecionado como de alta prioridade - mas deve-se utilizá-lo com caltela.

Há também o caso da utilização de um seletor mais específico, como um seletor "p", que altera as cores dos parágrafos, mesmo com uma cor já atribuída ao body. Um seletor mais específico terá precedência sobra um seletor mais geral. Um seletor de identidade (id), por exemplo, sempre terá prioridade em relação a um seletor de classe (class), por exemplo.
